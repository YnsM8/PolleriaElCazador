## Prioridades

1. Que funcione localmente.
2. Que cumpla la consigna.
3. Que la arquitectura sea clara.
4. Que el dashboard tenga buena narrativa.
5. Que el código sea simple y mantenible.

## Restricciones

- No implementar login.
- No usar APIs externas.
- No usar servicios de pago.
- No sobreoptimizar.
- No cargar datasets enormes completos al frontend.
- Mantener backend y frontend separados.
- Verificar cada fase antes de pasar a la siguiente.

## Criterio de terminado

El proyecto está terminado cuando:
- Backend corre sin errores.
- Frontend corre sin errores.
- La generación de datos funciona.
- Los KPIs se calculan desde datos reales generados.
- La regresión lineal entrena y predice.
- K-Means segmenta platos.
- El asistente IA responde.
- Existe README con comandos de ejecución.
- Existe una página o sección que cubre cada punto de la consigna.
